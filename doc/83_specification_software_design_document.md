# SW設計書の記載ルール

## 目的
SW設計の設計根拠を記録し、実装のブラックボックス化を防ぐ。
SW設計情報を残すことで設計手法を流用可能とする。

## 保存場所
- ファイルは、`doc/2*_***.md` とする。
- すでにファイルが存在する場合は内容を更新、または新規設計内容であれば適切な場所に追記する

## 設計書の種類
- **運用設計(21_operation_design_document.md)** : インストール方法、実行方法、デバッグ方法、テスト方法を記録する
- **機能設計(22_function_design_document.md)** : 機能面の設計結果を記録する
- **UX設計(23_ux_design_document.md)** : UI設計、画面設計、画面遷移設計を記録する
- **テーブル設計(24_table_design_document.md)** : テーブル設計を記録する
- **アルゴリズム設計(25_algorithm_design_document.md)** : 複雑な制御（数値計算、バッファ管理、デザインパターン等）方法を記録する
- **詳細設計(26_detail_design_document.md)** : 以下を記録する。
  - ディレクトリ、ファイル構成
  - ディレクトリ説明、ファイル説明
  - パラメータ一覧、概要、設定範囲、デフォルト値
  - ユースケース一覧、説明
  - 処理シーケンス(PlantUML形式のSequence図)
  - 処理フロー (PlantUML形式のActivity図)
  - 状態遷移  (PlantUML形式のState図)
  - 利用ライブラリ一覧、説明
